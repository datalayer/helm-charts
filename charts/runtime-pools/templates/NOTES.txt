Your runtime-pools server has now been installed, and can be accessed in the following ways:

1. Connect to the runtime-pools CLI pod:

  kubectl exec -n {{ .Release.Namespace }} $(kubectl get pod -n {{ .Release.Namespace }} --selector=app={{ template "runtime-pools.name" . }} -o jsonpath='{.items...metadata.name}') -- sh

2. Connect with HTTP internally, within the kubernetes cluster on:

  {{ template "runtime-pools.service-name" . }}.{{ .Release.Namespace }}:{{ .Values.runtime-pools.port }}

3. Connect with HTTP Node Port to the kubernetes cluster:

  export NODE_IP=$(kubectl get nodes --namespace runtime-pools -o jsonpath="{.items[0].status.addresses[0].address}")
  export NODE_PORT=$(kubectl get --namespace runtime-pools -o jsonpath="{.spec.ports[0].nodePort}" services {{ template "runtime-pools.service-name" . }})
  echo http://$NODE_IP:$NODE_PORT/runtime-pools/

4. Connect with HTTP external service to the kubernetes cluster:

  export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "app={{ template "runtime-pools.name" . }}" -o jsonpath="{ .items[0].metadata.name }")
  echo "Visit http://127.0.0.1:{{ .Values.runtime-pools.port }}/runtime-pools/ to access runtime-pools"
  kubectl port-forward --namespace {{ .Release.Namespace }} $POD_NAME {{ .Values.runtime-pools.port }}:{{ .Values.runtime-pools.port }}

5. For Minikube:

  open http://minikube.local/api/jupyter-server/
